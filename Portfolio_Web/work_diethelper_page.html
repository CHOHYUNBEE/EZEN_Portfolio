Development Review

미디어 쿼리를 이용한 유연한 콘텐츠 배치

해상도에 따른 사이즈 변화를 위해 px가 아닌 가변 가능한 vw, vh 사용

구현된 프로그램 기능 (css, javascript, jQery)

편의점 별 음식 data 파일을 불러와 json 파일로 변환 후 저장

프로그램 세부 설명서 클릭 시 class를 통해 보이고 가려지는 이벤트

사용자 개인 정보 입력 시 계산식에 따른 개인 영양정보 계산

원하는 편의점, 식사시간, 식단 타입 선택 시 개인 영양 성분에 따른 해당 편의점 음식 List에서 식단 (1~2가지) 자동 생성

식단 개수는 랜덤 함수를 이용하여 50% 확률로 만들어지게 작성

사용자 필요 영양 성분(칼로리, 탄수화물, 단백질, 지방)과 만들어진 식단의 영양성분(칼로리, 탄수화물, 단백질, 지방)의 비율을 color bar와 수치로 나타냄

칼로리 기준 탄수화물, 단백질, 지방의 비율을 계산식에 맞춰 conic-gradient를 통해 원형 그래프로 구현

제작된 식단 중 하나 선택 후 삭제 시 식단 리스트에서 삭제되고 총 칼로리 및 영양 성분 수정

식단 재생성 버튼 클릭 시 현재 존재하는 식단 칼로리 제외 후 식단 재생성 및 영양 성분 정보 수정

각 버튼 및 동작 과정별 상황에 맞는 confirm()과 alert() 등 피드백을 제공

식단 생성이 오래 걸릴 경우 animation을 이용해 로딩 화면을 구현하여 나타냄

Review

문제점 및 해결 과정

원하는 편의점 별 음식 Data API가 존재하지 않아 데이터 조회 문제
→ 칼로리가 작성된 사이트의 정보를 가져와 txt로 저장하면 데이터를 json 양식에 맞게 변화하여 파일이 자동 저장 되는 사이트 제작

javaScript에서 동적으로 생성된 요소에 이벤트 적용이 되지 않음
→ $(document)를 사용하여 시작점부터 모든 DOM 탐색을 거쳐서 하위단까지 객체를 탐색하여 이벤트 적용

식단 List에서 데이터를 불러오지 않아 Maximum call Error 발생
→ try catch 문을 활용하여 정상 동작 시 식단 생성, 에러 발생 시 식단 생성이 되지 않음을 표시해 줌

잘한 점

정해진 기간 내에 디자인, 개발, 테스트까지 모두 완료 하였으며 초기 구현 목표로 잡았던 부분을 모두 구현해 냄

기존에 없던 프로그램으로 디자인, 개인 정보 작성, 그에 맞는 영양성분부터 데이터 변환, 식단 생성 등 모든 과정에 대한 알고리즘을 직접 작성하고 개발함

이전의 개발 프로그램들보다 중복되거나 자주 사용되는 로직들은 함수로 만들어 불필요한 코드를 줄여냄

계산식이 많고 js를 통해 새롭게 데이터들을 구하고 나타내는 과정이 많았지만 계산되는 과정을 잘 정리하여 표현해냄

아쉬운 점

디자인적인 요소가 조금 아쉬움. 프로젝트 종료 후에도 디자인에 대해 조금 더 생각해 내어 수정할 예정

식단 데이터가 편의점 음식에 국한되어 있다 보니 다양한 식단 리스트가 출력되지 않는 것이 아쉬웠음

사용자의 영양성분에 근접하게 식단을 생성하지만 오차 범위가 존재함 (오차를 줄이면 식단이 다양하지 않아 식단이 생성되지 않는 문제가 존재)
